# 14日整理-放量突破-缩量回踩 形态策略说明书

这份说明书的目标，不是描述“看起来像不像”，而是把一个短线盘感模型改写成可以直接回测、编程、复核的规则版本。

原始口述版本：
- 回看14个交易日
- 前10个交易日最高收盘价和最低收盘价差值6%以内
- 第11个交易日阳线上涨7%以上，量能放大1倍以上
- 第12-14个交易日缩量跌，每日跌幅＜3%

上面这段话有几个典型歧义：
1. “回看14日”是包含当日还是不包含当日
2. “量能放大1倍”是相对前1日、前5日均量还是前10日均量
3. “缩量跌”是必须3天都收阴，还是允许小阳/十字星
4. “最高收盘价和最低收盘价差值6%以内”是否只看收盘，不看盘中振幅

下面给出严格版定义。

--------------------------------------------------
一、时间窗口定义
--------------------------------------------------

以某个交易日 t 作为“候选确认日”。

向前取连续14个交易日：
- d1 = t-13
- d2 = t-12
- ...
- d10 = t-4
- d11 = t-3
- d12 = t-2
- d13 = t-1
- d14 = t

也就是说：
- d1-d10 = 整理期
- d11 = 突破日
- d12-d14 = 回踩整理期

所有条件都以这14根日K为准。

--------------------------------------------------
二、所需字段
--------------------------------------------------

每个交易日至少需要：
- 日线 open
- 日线 high
- 日线 low
- 日线 close
- 日线 volume
- 日线 amount（可选）
- 日线 turnover_rate（建议有，用于过滤僵尸股）
- 周线 open / high / low / close
- 周线 MA5 / MA13 / MA21（如果没有预先算好，可由周线收盘价滚动计算）

--------------------------------------------------
三、严格版基础规则
--------------------------------------------------

先加你新补充的“中期趋势过滤”。

A. 周线趋势过滤

规则 W1：周K处于上升趋势
以信号日 t 所在周记为 w0，向前回看最近 8 根已完成周K：
- weekly_ma21_slope = MA21[w0-1] / MA21[w0-4] - 1
- 要求：weekly_ma21_slope > 0

更直观的替代写法：
- MA21[w0-1] > MA21[w0-2] > MA21[w0-3]

解释：
- 你说的“周K上升趋势”不能只靠肉眼说
- 这里把它落成“周线慢均线向上”
- 用 MA21 来定义趋势，比直接看单根周K稳一点

规则 W2：周线多头排列
- MA5[w0-1] > MA13[w0-1] > MA21[w0-1]

解释：
- 这是最直接的“周K多头排列”定义
- 只用已完成周K，避免把本周未走完的数据误算进去

规则 W3：5 / 13 / 21 周均线要呈“高-中-低”上移结构
- high_band = MA5[w0-1]
- mid_band = MA13[w0-1]
- low_band = MA21[w0-1]
- 要求：high_band > mid_band > low_band
- 且：MA5[w0-1] > MA5[w0-2]
- 且：MA13[w0-1] >= MA13[w0-2]
- 且：MA21[w0-1] >= MA21[w0-2]

解释：
- 你说的“5 13 21日线高中低排列往上”，放到趋势过滤里更合理地解释为：
  - 快线在上
  - 中线居中
  - 慢线在下
  - 三条线整体向上或至少不走坏
- 因为你前面又说“周K上升趋势”，所以这里我按“周线 MA5/13/21”写成正式版
- 如果你本意其实是“日线 MA5/13/21”，那可以再单独加一条日线趋势过滤版

规则 W4：当前日线价格不能明显弱于周线结构
- close[d14] >= MA13[w0-1]

解释：
- 如果周线看着多头排列，但当前日线已经明显跌回周13周线下方，说明形态质量要打折
- 这条建议先做为增强过滤，不一定一上来强制启用

B. 整理期：d1-d10

规则 A1：收盘价窄幅整理
- close_range_10 = max(close[d1:d10]) / min(close[d1:d10]) - 1
- 要求：close_range_10 <= 0.06

解释：
- 这保留了你朋友原始逻辑
- 只看收盘价，不把盘中噪音算进去

规则 A2：平台内部不能太散
- 每日绝对涨跌幅：abs(close[i] / close[i-1] - 1)
- 在 d2-d10 内，要求 max_daily_abs_ret_10 <= 0.04

解释：
- 如果整理期里某一天已经大涨大跌 6%-8%，那就不是“窄幅整理”了
- 这里加这个条件，是为了防止只靠收盘价范围筛出假平台

规则 A3：可选增强版，避免盘中振幅过大
- intraday_range_10 = max(high[d1:d10]) / min(low[d1:d10]) - 1
- 建议要求：intraday_range_10 <= 0.10

解释：
- 这是增强条件，不是最小必要条件
- 如果你想保留更多候选，可以先不启用

规则 A4：流动性下限
- avg_turnover_10 = mean(turnover_rate[d1:d10])
- 如果有换手率数据，建议要求：avg_turnover_10 >= 0.5%
- 如果没有换手率，可以改为 avg_amount_10 >= 自定义阈值

解释：
- 这是为了防止筛出成交极差的票

C. 突破日：d11

规则 B1：d11 必须收阳
- close[d11] > open[d11]

规则 B2：d11 单日涨幅足够强
- breakout_ret = close[d11] / close[d10] - 1
- 原始严格版要求：breakout_ret >= 0.07

说明：
- 这是完全保留你朋友原始条件的写法
- 若未来要做优化版，可测试 >= 0.05 或 >= 0.06

规则 B3：d11 放量
- vol_ratio_5 = volume[d11] / mean(volume[d6:d10])
- vol_ratio_10 = volume[d11] / mean(volume[d1:d10])
- 严格版要求：vol_ratio_5 >= 2.0

解释：
- “放大1倍以上”最容易歧义
- 这里明确写成：相对前5日平均成交量，至少放大到 2 倍
- 因为“放大1倍”如果按中文交易语境，通常等于“变成原来的2倍”

规则 B4：d11 收盘必须突破整理平台上沿
- resistance_close_10 = max(close[d1:d10])
- 要求：close[d11] > resistance_close_10

可选更强版本：
- close[d11] >= resistance_close_10 * 1.01

解释：
- 不然只是大阳线，不一定是平台突破

规则 B5：d11 不可长上影失败
- upper_shadow_ratio = (high[d11] - max(open[d11], close[d11])) / max(close[d11] - open[d11], 0.0001)
- 建议要求：upper_shadow_ratio <= 0.5

解释：
- 防止冲高回落、资金承接差的假突破
- 这是增强条件，可选

D. 回踩整理期：d12-d14

这是原始口述里最容易写歪的部分，所以必须拆严。

规则 C1：三天都不能出现大跌
对 i in {d12, d13, d14}：
- daily_ret_i = close[i] / close[i-1] - 1
- 要求：daily_ret_i > -0.03

说明：
- 原始口述是“每日跌幅 < 3%”
- 严格写法就是单日跌幅不超过3%

规则 C2：三天总体是回踩，而不是崩掉
- pullback_ret_3d = close[d14] / close[d11] - 1
- 要求：pullback_ret_3d >= -0.06

解释：
- 三天总回撤不能太大
- 否则不是健康回踩，而是突破失败

规则 C3：回踩期缩量
- pullback_avg_vol = mean(volume[d12:d14])
- 要求：pullback_avg_vol <= 0.70 * volume[d11]

解释：
- 直接把“缩量”写成：回踩三天平均量 <= 突破日成交量的70%
- 这是最容易编程、也最贴近盘感的写法

规则 C4：回踩不跌回整理平台下方太多
- support_close_10 = min(close[d1:d10])
- 建议要求：close[d14] >= support_close_10

更推荐的版本：
- close[d14] >= resistance_close_10 * 0.98

解释：
- 如果突破后3天就重新跌回平台深处，通常质量较差

规则 C5：允许“弱整理”，不强制三天都收阴
- 不要求 d12-d14 三天必须全部收阴
- 只要求：
  - 没有大跌
  - 总回撤受控
  - 量能收缩

解释：
- 这是很重要的修正
- 真实强票经常是：
  - 一天小跌
  - 一天十字星
  - 一天小阳/小跌
- 如果强制“三天都跌”，反而会漏掉很多高质量形态

--------------------------------------------------
四、最小可执行版本（推荐先用于回测）
--------------------------------------------------

如果你要先做一个最简洁、最稳的版本，我建议先用下面这套。

最小执行规则：

1. 周线趋势过滤
- MA5[w0-1] > MA13[w0-1] > MA21[w0-1]
- MA5[w0-1] > MA5[w0-2]
- MA13[w0-1] >= MA13[w0-2]
- MA21[w0-1] >= MA21[w0-2]

2. 整理期 d1-d10
- max(close[d1:d10]) / min(close[d1:d10]) - 1 <= 0.06
- max(abs(close[i] / close[i-1] - 1) for i in d2:d10) <= 0.04

3. 突破日 d11
- close[d11] > open[d11]
- close[d11] / close[d10] - 1 >= 0.07
- volume[d11] / mean(volume[d6:d10]) >= 2.0
- close[d11] > max(close[d1:d10])

4. 回踩期 d12-d14
- 对 d12,d13,d14：close[i] / close[i-1] - 1 > -0.03
- close[d14] / close[d11] - 1 >= -0.06
- mean(volume[d12:d14]) <= 0.70 * volume[d11]

--------------------------------------------------
五、策略输出定义
--------------------------------------------------

某只股票在 t 日满足全部规则时：
- signal_date = t
- pattern_name = breakout_pullback_14d_v1
- signal = 1

否则：
- signal = 0

建议额外输出以下诊断字段：
- weekly_ma5
- weekly_ma13
- weekly_ma21
- weekly_ma21_slope
- weekly_bull_alignment = (MA5 > MA13 > MA21)
- close_range_10
- max_daily_abs_ret_10
- breakout_ret
- vol_ratio_5
- pullback_ret_3d
- pullback_avg_vol_ratio = mean(volume[d12:d14]) / volume[d11]
- close_above_platform_ratio = close[d11] / max(close[d1:d10]) - 1

这样后面能直接看每个条件到底卡在哪。

--------------------------------------------------
六、入场与验证定义
--------------------------------------------------

这个形态本身只是“候选信号”，不是完整交易系统。

如果要回测，必须额外定义：

1. 入场时间
推荐两种：
- 方案A：d14 收盘后记为信号，下一交易日开盘买入
- 方案B：d14 收盘即记为理论入场价（用于简化回测）

建议默认使用：
- 下一交易日开盘买入

2. 持有周期
先固定测试：
- 持有 1 / 3 / 5 / 10 个交易日

3. 退出方式
第一版先不用复杂止盈止损，先看：
- future_ret_1d
- future_ret_3d
- future_ret_5d
- future_ret_10d

第二版再考虑：
- 跌破 d11 最低价止损
- 跌破平台上沿止损
- 固定收益止盈

--------------------------------------------------
七、容易误判的情况
--------------------------------------------------

下面这些形态，表面上像，实战里往往质量一般：

1. d11 虽然大涨，但上影过长
- 容易是假突破

2. d12-d14 看似缩量回踩，但实际上已经跌回平台深处
- 说明承接不够

3. 整理期价格范围虽然不大，但每天上下影乱飞
- 说明并不是真正的稳定蓄势

4. 突破日是消息刺激一字/情绪冲顶
- 后续很容易衰减

5. 流动性太差
- 图形再好也可能无法交易

--------------------------------------------------
八、建议的优化方向（先不要一上来就加到规则里）
--------------------------------------------------

等你回测完基础版，再看是否加入：

1. 相对市场强度过滤
- d11 当日超额收益 > 某阈值

3. 日线 5 / 13 / 21 均线过滤（如果你后续确认本意是日线而不是周线）
- MA5_d > MA13_d > MA21_d
- 且三条线最近2-3日持续上行

4. 回踩位置过滤
- d14 收盘仍在 d11 阳线实体上半部

5. 题材/公告叠加
- 只保留最近7日有正向事件催化的标的

这几条更适合做第二版增强。

--------------------------------------------------
九、最终推荐命名
--------------------------------------------------

建议把这套模型命名成：
- breakout_pullback_14d_v1

如果以后有优化版：
- breakout_pullback_14d_v2
- breakout_pullback_14d_with_event

--------------------------------------------------
十、一句话总结
--------------------------------------------------

这套策略的严格版定义可以概括为：

“先要求周线处于上升趋势并满足5/13/21均线多头排列，再找10日收盘窄幅整理的平台，再找1根放量突破阳线，最后要求3日内用较小回撤和明显缩量完成回踩；满足后在回踩完成日记为候选信号。”
